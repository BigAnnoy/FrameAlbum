## 🎯 迁移策略

### 渐进式迁移原则

1. **向后兼容优先**：旧代码保持可用，不强制立即迁移
2. **模块共存**：新旧代码可以同时工作
3. **按模块迁移**：逐个模块迁移，降低风险
4. **充分测试**：每个模块迁移后进行充分测试

### 兼容性层工作原理

```javascript
// 兼容性层 (compatibility.js)
import { utils, api } from './index.js';

// 暴露全局变量
window.API = {
    getAlbumStats: api.album.getAlbumStats,
    // ... 其他 API 映射
};

window.APIUtils = {
    formatFileSize: utils.formatFileSize,
    // ... 其他工具映射
};
```

这样，旧代码可以继续使用 `window.API` 和 `window.APIUtils`，而新代码可以直接导入模块使用。

## 💡 使用指南

### 新代码写法

```javascript
// 导入模块使用
import { utils, api, app } from './modules/index.js';

// 使用工具函数
const size = utils.formatFileSize(1024);

// 调用 API
const stats = await api.album.getAlbumStats();

// 使用 App 类
const myApp = new app.App();
```

### 旧代码写法（仍可用）

```javascript
// 继续使用全局变量（通过兼容性层）
const size = APIUtils.formatFileSize(1024);
const stats = await API.getAlbumStats();
```
